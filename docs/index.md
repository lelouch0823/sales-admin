# 1. 背景与目标

管理端用于内部运营/门店管理人员完成以下工作：
- 管理“今日推荐”手动配置（全局与门店级），并可预览 App 端最终生效结果。
- 管理客户数据（含公海客户池、客户领取/回收、共享协作成员）。
- 管理用户与权限（创建用户时设定角色/权限）。
- 提供审计日志与关键操作记录，确保内部系统可追溯（推荐配置、客户归属变更、共享权限调整等）。

# 5. 非功能需求（NFR）

### 5.1 安全
- 全站必须认证（JWT/SSO）
- 后端每个接口做 RBAC 校验
- 防止越权：tenant 级隔离必须在查询层就约束（而不是前端隐藏）

### 5.2 可用性与体验
- 列表页支持分页、过滤、排序（避免一次加载过多）
- 所有写操作有明确成功/失败提示，失败给出可理解的错误信息
- 支持“保存中”禁用按钮、防止重复提交

### 5.3 性能
- 客户列表/公海池分页查询：单页 50 以内响应 < 500ms
- 推荐预览：limit 12/20 响应 < 300ms
